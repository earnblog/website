# 成稿自检清单(内部用,不展示给用户)

出稿前对照。**凑齐 ≥7 条才放行**,不够就改。量化文体例外(它本就克制,见末尾)。

1. [ ] 标题是反直觉断言或重定义,且带感叹号"!"
2. [ ] 开头是三选一(首句即论点 / 三连"不是A而是B" / 立靶)之一
3. [ ] 有一句核心命题,全文复现 ≥3 次
4. [ ] 标题、核心命题、末句金句三者同义复沓
5. [ ] 用了脊柱(自发秩序 / 反身性正反馈 / 算力 / 信用)解释现象
6. [ ] 有一段正反对称 或 早期/末期镜像 推演
7. [ ] 至少出现一个"不是A,而是B"句式
8. [ ] 末句是独立成段的金句
9. [ ] 成语/对仗密度够(全文 ≥4 处)
10. [ ] 有转折锚词(实际上/归根到底/本质上/从这个意义上讲)
11. [ ] 升维收尾——拉到算力/人性/秩序/谦卑(量化文体免)
12. [ ] 文末挂了免责声明
13. [ ] 通过政治硬过滤(无政治内容)
14. [ ] 没有照搬贝乐斯付费文的原句

## 质量否决项(命中任一,必须重写)

- 写成了"正确但没靶子"的空泛文(没有具体现象/观点可拆)
- 核心命题不够"重定义"(只是普通观点,没把常识翻过来)
- 堆了金句但没有真推演(光升维不跑机制)
- 编造了不存在的数据/指标
- **数字没有当场翻译**:出现了关键数字或专业术语,却没有紧跟一句大白话(见 style-rules.md「数字与术语的写法」)
- **数字在做证明而非论证**:摆了一串核实过的数字,但砍掉不影响论证——说明它只是在展示"我查过"。单篇关键数字超 8 个,或某段超 3 个,直接砍

## 量化文体特例

量化硬核文(LPPL/凹凸/Sigma)**不要求**第4、8、11条(同义复沓、金句、升维)。它要的是:真实数据、冷静克制、技术准确、不抒情。命中"编造数据"否决项直接毙。没有真实数据就不写这个文体。
